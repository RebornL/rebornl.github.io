Java性能优化权威指南

2019-01-25

第一章：策略、方法和方法论

Wilson和Kesselman添加性能测试分析之后的软件开发流程

性能分析的两种方法

  • 自顶向下：从软件栈顶层的应用，从上往下寻找优化机会和问题。第一步对运行在特定负载之下的应用进行监控（监控操作系统资源，Java虚拟机，Java EE容器以及应用的性能测量统计指标）；第二步根据监控的信息进行下一步的工作，JVM垃圾收集器调优，JVM命令行调优等等。
  • 自底向上：从软件栈最底层的CPU统计数据开始，上升到应用自身的结构或者使用方式。更侧重于关注CPU的使用。

第二章：操作系统性能监控

改善性能涉及三个方面：

  • 性能监控：非侵入方式手机或查看应用运行性能数据的活动
  • 性能分析：侵入式收集运行性能数据的活动，会影响应用的吞吐量或响应性
  • 性能调优：是一种为改善应用响应或吞吐量而参数、源代码或属性配置的活动

Linux上查看CPU使用率的命令有：vmstat，mpstat，top等等。

CPU调度程序运行队列对于分辨系统是否满负荷也有重要的意义。

内存利用率，在系统进行页面交换或使用虚拟内存是，Java程序或JVM会表现出明显的性能问题。

网络I/O使用率，Linux上可以使用nicstat

磁盘I/O使用率，Linux上可以使用iostat
