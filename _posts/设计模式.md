# 设计模式

## 创建型模式

### 单例模式

饿汉式：全局单例在类装载时构建

懒汉式：全局的单例在第一次使用时再创建

```java
//饿汉式，线程安全
public class Singleton {
    private static Singleton uniqueInstance = new Singleton();
    private Singleton() {}
    public static Singletion getInstance() {
        return uniqueInstance;
    }
}

//懒汉式,双重检测加锁
public class Singleton {
    private volatile static Singletion uniqueInstance;
    private Singleton() {}
    public static Singleton getInstance() {
        if (uniqueInstance == null) {
            synchronized(Singleton.class) {
                if (uniqueInstance == null) {
                    uniqueInstance = new Singleton();
                }
            }
        }
    }
    return uniqueInstance;
}
```

### 适配器模式

适配器很简单，举个常见的例子，香港和大陆的电器插头是不一样的，当在香港买的电器在大陆使用的话，这个时候就需要一个插头转换器将香港的插头转换成大陆可用的插头。适配器就是将类的接口转换成客户期待的接口样子。

JDK中用到适配器模式的地方：

```java

java.io.InputStreamReader(InputStream)
java.io.OutputStreamReader(OutputStream)
```

# 操作系统

特征：并发（与并行概念的区别），共享，虚拟，异步

功能：进程管理，内存管理，文件管理，设备管理

内核态和用户态的区别：当进程或者任务执行系统调用而陷入内核代码中执行时，就称进程处于内核态，此时CPU位于特权级最高的内中代码中执行。当进程执行自身的代码，称其处于用户态。<https://blog.csdn.net/youngyoungla/article/details/53106671>

中断类型：外中断（IO完成终端，时钟中断，控制台中断），异常，陷入



进程：资源分配基本单元，进程控制块（PCB，描述进程基本信息和运行状态）

线程：独立调度基本单位，同一进程内的多个线程共享进程资源

两者的区别：1. 拥有资源；2. 调度；3. 系统开销；4. 通信



进程三种状态：就绪状态，运行状态和阻塞状态



进程调度算法：

批处理系统：保证吞吐量和周转时间

- 先来先服务
- 短作业优先
- 最短剩余时间

交互式系统：追求快速响应

- 时间片轮转
- 优先级调度
- 多级反馈队列



进程通信IPC

- 匿名管道：只允许父子间进程通信，半双工
- 命名管道：解除只允许父子间进程通信的问题
- 消息队列
- 信号量
- 共享内存
- 套接字Socket



死锁的四个必要条件：

- 互斥
- 占有和等待
- 不可抢占
- 环路等待



死锁的处理方式：

- 鸵鸟策略：不处理，出现的概率很低
- 死锁检测：根据资源有向图判断环是否存在，从而判断是否有死锁。
- 死锁恢复：抢占恢复（允许一个资源被多个任务使用）；回滚回复；杀死进程回复（破坏环路等待条件）
- 死锁预防：破环四个必要条件
- 死锁避免：银行家算法



虚拟内存：将物理内存扩充成更大的逻辑内存。虚拟内存管理，会将地址空间划分成固定大小的一块一块，称之为一页，程序执行时，会将对应的页映射（硬件执行必要的映射）到物理内存中。



页面置换算法：

- 最佳置换算法（OPT）：将最长时间不再被访问的页面换出，以获得最低的缺页率
- 最近最久未使用（LRU）：这是常用的页面置换算法，容易实现，将最近最久未被使用的页去掉
- 最近未使用（NRU）：设置两个状态位R（被访问置1）和M（被修改置1）
- 先进先出算法（FIFO）
- 第二次机会算法：结合FIFO和NRU算法



分段：解决分页的固定大小情况下，动态增长的倒置覆盖的问题。每个段有独立的地址空间，长度可以不同，并且动态增长。分段主要是为使程序和数据可以划分成逻辑上独立的地址空间有助于共享和保护。



文件系统：主要组成：inode+block

inode：一个文件占用一个inode，记录文件属性和所在block编号

block：记录文件内容，文件太大时，会占用多个block

Ext2文件系统，读取文件时，先从inode中查找文件内容所有的block，再从block中读取内容

Fat文件系统，没有inode，而是每个block中存储着下一个block的编号



# MySQL

ACID:事务是一组满足ACID特性的一组操作。

- A：原子性
- C：一致性
- I：隔离性
- D：持久性



并发性问题：其实都是一致性不满足（隔离性没有做好）

- 丢失更改：两个事务对同一数据进行修改，T1修改，T2随后修改，T2覆盖T1的修改。可以通过行加锁解决这个问题。
- 脏读：T1事务对某一数据进行修改，T2可以读取到T1修改后的数据，随后T1撤销此次修改，那T2读取到的数据则是脏数据。可通过**提交读**这一隔离级别解决
- 不可重复读：T1事务范围内两次相同的查询返回了不同的数据，原因是T2事务对此数据做了**修改**导致的。**可重复读**的隔离级别可以解决这个问题
- 幻读：T1以相同条件查询以前检索过数据，发现返回的结果不同，原因在于T2事务进行了**插入和删除**了某些数据，因此第二次的检测结果和第一次不同。可以通过**串行读**这个隔离级别避免这个问题。在MySQL中InnoDB引擎中MVCC和Next-key Locks避免幻读的问题。



数据库的锁机制：

- 表锁：MyISAM引擎使用的就是表锁，开销小，加锁快，无死锁，锁定粒度大，发生冲突的概率搞，并发度最低。MyISAM的读写锁调度，以写优先，这也是MyISAM不适合做写为主表的引擎。
- 行锁：InnoDB引擎使用的加锁方式，开销大，加锁慢（因为更精细），容易出现思路，粒度小，冲突概率低，并发度最高。
- 以上两种是根据对数据操作的颗粒度，从数据操作的类型读和写，还可以分成读写锁
  - 读锁（共享锁）：针对同一份数据，多个读操作可以同时进行而不会相互影响
  - 写锁（排他锁）：当前写操作没有完成之前，会阻塞其他写锁和读锁。





# Java中NIO

## 缓冲区

Buffer类，用于存储数据，channel之间数据传递便是通过缓冲区操作。

重要的几个属性：

- 容量（Capacity）：缓冲区能够容纳数据元素的最大数量
- 上界（Limit）：在可读或可写状态切换之间，缓冲区第一个不能被读或写的索引位置
- 位置（Position）：下一个要被读或写的元素的索引
- 标记（Mark）：备忘位置，通过mark()设定mark = position，reset()设定position = mark
